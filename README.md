# gost-34.10-2012
Лабораторная работа, Схемы электронной подписи, ЭЦП
ГОСТ Р 34.10‑2012 – Электронная цифровая подпись на эллиптических кривых


Реализация алгоритмов формирования и проверки электронной цифровой подписи (ЭЦП) в соответствии с ГОСТ Р 34.10‑2012 – российским стандартом на основе эллиптических кривых.

Программа полностью написана на Python 3 без использования внешних криптографических библиотек (кроме стандартных модулей). Предназначена для образовательных целей – изучения принципов работы асимметричных криптосистем на примере ЭЦП.

🚀 Функциональные возможности
№	Опция	Описание
1	Генерация ключевой пары	Создаётся секретный ключ d (случайное из [1, q-1]) и открытый ключ Q = d·G. Ключи автоматически сохраняются в папку gost.
2	Подпись сообщения (ввод с клавиатуры)	Пользователь вводит текст, программа вычисляет подпись (r, s). Сообщение и подпись автоматически сохраняются.
3	Проверка подписи (оригинал)	Верификация подписи для того же сообщения – должна быть успешной.
4	Проверка подписи (изменённое)	К сообщению добавляется (ИЗМЕНЕНО) – подпись должна стать недействительной.
5	Показать информацию	Вывод параметров кривой, текущих ключей, сообщения и подписи.
6	О программе	Справка по стандарту и математическим формулам.
7	Управление файлами	Подменю для сохранения/загрузки ключей, сообщений, подписей, а также выбора произвольных файлов.
Особенности:

🎨 Цветной вывод в терминале (поддержка Windows через ctypes).

📁 Автоматическая работа с файлами – все данные сохраняются в папку C:\Users\kholostykh.iuv\Desktop\gost (создаётся при первом запуске).

🔧 Гибкая загрузка – можно указать произвольный путь к файлу сообщения или подписи.

📐 Пошаговый вывод математических вычислений (λ, x₃, y₃, бинарный алгоритм).

💻 Требования
Python версии 3.8 или выше.

🔧 Установка и запуск
Скачайте файл gost_sign.py из репозитория:

bash
git clone https://github.com/yvkholostykh/gost-34.10-2012.git
cd gost-34.10-2012
Запустите программу:

bash
python gost_sign.py
При первом запуске автоматически создастся папка gost на рабочем столе (путь можно изменить в коде). Все файлы ключей, сообщений и подписей будут сохраняться туда.

📖 Инструкция по использованию
Главное меню
text
----------------------------------------------------------------------
                         ГЛАВНОЕ МЕНЮ
----------------------------------------------------------------------
 1. Сгенерировать ключевую пару (автосохранение)
 2. Подписать сообщение (ввод с клавиатуры, автосохранение)
 3. Проверить подпись (по текущим данным в памяти)
 4. Проверить подпись (изменённое сообщение, по памяти)
 5. Показать информацию
 6. О программе (ГОСТ и формулы)
 7. Управление файлами (сохранить/загрузить/проверить)
 0. Выход
----------------------------------------------------------------------
Порядок действий для успешной проверки
Выберите 1 – сгенерировать ключевую пару.

Выберите 2 – введите сообщение (например, Привет, мир!). Программа подпишет его и покажет (r, s).

Выберите 3 – проверить подпись для оригинального сообщения → результат ПОДПИСЬ ВЕРНА.

Выберите 4 – программа сама изменит сообщение (добавит (ИЗМЕНЕНО)) и проверит → результат ПОДПИСЬ НЕВЕРНА.

Управление файлами
При выборе пункта 7 открывается подменю:

text
--- УПРАВЛЕНИЕ ФАЙЛАМИ ---
   Рабочая папка: C:\Users\kholostykh.iuv\Desktop\gost
   Ожидаемые имена файлов (в рабочей папке):
      secret_key.txt   – секретный ключ (одно число)
      public_key.txt   – открытый ключ (x и y по строкам)
      message.txt      – сообщение (UTF-8)
      signature.txt    – подпись (r и s по строкам)

   Доступные действия:
      a. Сохранить ключи (в рабочую папку)
      b. Сохранить сообщение (в рабочую папку)
      c. Сохранить подпись (в рабочую папку)
      d. Загрузить подпись и проверить (с указанием путей)
      e. Загрузить открытый ключ (из рабочей папки)
      f. Выбрать файл сообщения (произвольный путь)
      g. Выбрать файл подписи (произвольный путь)
      0. Назад
a, b, c – сохраняют текущие данные в стандартные файлы.

d – интерактивно запрашивает пути к файлам сообщения, подписи и открытого ключа, затем выполняет проверку.

e – загружает открытый ключ из public_key.txt (рабочая папка).

f, g – позволяют выбрать произвольный файл для сообщения или подписи.
